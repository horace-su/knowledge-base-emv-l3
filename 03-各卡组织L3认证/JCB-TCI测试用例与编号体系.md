# JCB TCI 测试用例分组与编号体系

> 在 [JCB 与 UnionPay L3 认证深度解析](./JCB与银联-L3认证深度解析.md) 的 TCI 框架之上，进一步拆 **TCI（J/Smart，接触）与 TCI-CL（J/Speedy，非接）的测试用例分组、覆盖维度与编号体系**。承接 [各卡组织 L3 认证测试要求一览](./各卡组织L3认证测试要求一览.md)、[FIME-BTT .tpp 与 L3 测试计划](../02-fime测试工具/FIME-BTT-TPP项目文件与L3测试计划.md)。

## 一、TCI 在 JCB 认证链中的位置

| 层级 | 内容 | 与 TCI 关系 |
|------|------|-------------|
| **L1** | 物理/电气（接触 ISO 7816、非接 ISO 14443） | EMVCo Type Approval，TCI 之前 |
| **L2** | 内核（J/Smart 接触内核、J/Speedy 非接内核） | EMVCo + JCB 型式批准，TCI 前提 |
| **TCI / TCI-CL** | **终端集成实现检查（L3 等价）** | 本文焦点：终端+收单实现的端到端验证 |
| 主机认证 | 收单主机报文/清算 | 见 [收单主机认证](../06-报文层/收单主机认证与L3重测触发条件.md) |

- **TCI**（Terminal Check for Implementation）= 接触，验 **J/Smart**。
- **TCI-CL**（Contactless）= 非接，验 **J/Speedy**；非接还需 Type Approval 前提。
- 定位等同其它卡组织的 L3：是 JCB 的"品牌自有 QA"，部署前强制，与 EMVCo L1/L2 解耦。

## 二、用例分组（覆盖维度框架）

TCI/TCI-CL 用例按 EMV 交易生命周期组织，与其它卡组织 L3（如 [Visa ADVT/CDET](../04-visa专题/Visa-CDET非接测试用例细分.md)、[M-TIP TSE](../05-mastercard专题/M-TIP-TSE配置详解.md)）的维度同构：

### 2.1 接触（TCI / J/Smart）

| 分组 | 覆盖点 |
|------|--------|
| **应用选择** | PSE/直接选择、候选 AID 优先级、AID 部分匹配、多应用选择 |
| **离线数据认证 (ODA)** | SDA / DDA / CDA；CAPK 索引定位；证书链恢复（见 [ODA 字节级](../01-基础概念/ODA证书链字节级.md)） |
| **处理限制** | 应用版本、有效期、AUC 应用用途控制 |
| **持卡人验证 (CVM)** | 离线明文/密文 PIN、联机 PIN、签名、No CVM；CVM 列表回退（见 [接触与非接 CVM 详解](../01-基础概念/接触与非接CVM详解.md)） |
| **终端风险管理** | 楼层限额、随机选择、速度检查 |
| **终端行为分析 (TAA)** | TVR/TAC/IAC 评估、GENERATE AC 决策（见 [TAC/IAC/TVR](../01-基础概念/TAC-IAC-TVR决策逻辑.md)） |
| **联机处理** | ARQC 生成、ARPC 验证、发卡行脚本 `71`/`72`、第二次 GENERATE AC |
| **完成/异常** | 批准/拒绝/冲正、撤销、批上送 |

### 2.2 非接（TCI-CL / J/Speedy）

| 分组 | 覆盖点 |
|------|--------|
| **非接应用选择** | PPSE (`2PAY.SYS.DDF01`)、Kernel 指示、组合选择 |
| **非接交易流程** | GPO + PDOL、Fast Path、快速读记录 |
| **非接 CVM** | No CVM / 签名 / 联机 PIN / **CDCVM**（移动设备/Token） |
| **非接限额** | 楼层限额、CVM 限额、非接交易上限（见 [接触与非接 CVM 详解](../01-基础概念/接触与非接CVM详解.md)） |
| **非接 ODA** | fDDA / CDA（非接联机数据认证） |
| **令牌/移动** | Token、PAR、设备端 CDCVM 指示 |

## 三、编号体系结构（如何读 TCI 用例号）

> 与 [FIME-BTT .tpp 与 L3 测试计划](../02-fime测试工具/FIME-BTT-TPP项目文件与L3测试计划.md) 中其它卡组织编号体系并列理解。JCB 测试包的用例标识通常由以下要素组合而成（**结构性描述，非具体编号**）：

```
[程序前缀] - [接触/非接] - [功能分组] - [序号]
   TCI         /  CL          ODA          nn
```

| 编号要素 | 取值示意 | 说明 |
|----------|----------|------|
| 程序前缀 | `TCI` / `TCI-CL` | 接触 vs 非接 |
| 功能分组 | 应用选择 / ODA / CVM / 联机 / 脚本… | 对应 §二的维度 |
| 序号 | 数字 | 同分组内顺序 |

- 实际测试包里每条用例含：**前置条件（测试卡/参数）、操作步骤、期望结果（TVR/TSI 位、报文字段、终端显示）**——这与 [M-TIP TSE 问卷](../05-mastercard专题/M-TIP-TSE问卷与终端配置参数.md) 驱动选例的逻辑一致：先答终端能力问卷，工具据此裁剪适用用例集。
- 期望结果的校验锚点与其它卡组织相同：**TVR/TSI 字节、DE55 标签**（见 [DE55 逐标签清单](../06-报文层/ISO8583-DE55-逐标签实现清单.md)）、GENERATE AC 决策。

## 四、JCB 专有数据元注意点

| 标签 | 在 JCB 语境 | 注意 |
|------|-------------|------|
| `9F53` | **Terminal Interchange Profile**（JCB 部分用法） | ⚠️ 与通用 EMV 的 Transaction Category Code **标签复用冲突**——见 [DE55 逐标签清单 §四](../06-报文层/ISO8583-DE55-逐标签实现清单.md#四9f53-标签复用陷阱跨卡组织实现冲突点) |
| AID | JCB RID `A000000065` | 见 [AID 与 CAPK 全卡组织参考表](../01-基础概念/AID与CAPK全卡组织参考表.md) |
| CAPK | 随 JCB 测试包下发 | 测试卡用测试 CAPK，生产用生产 CAPK |

## 五、与其它卡组织的关系（一句话定位）

- TCI 是 **JCB International 单一运营**的程序（不像银联有独立国内体系，见 [JCB 与银联深度解析 §一](./JCB与银联-L3认证深度解析.md)）。
- 用例**维度**与 Visa/MC 的 L3 同构，**差异在专有数据元（如 `9F53` TIP）与 J/Smart、J/Speedy 内核行为细节**。
- 工具侧：JCB International 分别认可接触/非接的 TCI 测试工具，FIME BTT 等品牌测试平台可承载（见 [FIME-BTT 品牌测试工具](../02-fime测试工具/FIME-BTT-品牌测试工具深度解析.md)）。

> **需进一步落地**：精确用例编号与数量须申领 JCB 测试包后核对；本文框架可直接对照测试包逐条填充。

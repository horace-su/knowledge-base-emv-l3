# 各卡组织 L3 认证测试要求一览

> 本文梳理 BTT 所支持的 6 大卡组织各自的 EMV Level 3（终端集成/部署）认证程序、专属工具、测试计划与前置条件。
> 关联文档：[FIME-BTT 品牌测试工具深度解析](../02-fime测试工具/FIME-BTT-品牌测试工具深度解析.md) · [FIME L3 测试工具总览](../02-fime测试工具/FIME-L3测试工具深度解析.md)

## 通用前提（所有卡组织共通）

L3 测试发生在 L1（物理层）、L2（内核/支付应用）通过 EMVCo 认证之后，用于验证 **终端与收单主机及支付网络的端到端集成**。共通前提：

- 终端必须已获 **EMVCo L1/L2 批准**（内核/IFM 须为当前有效的 EMVCo approval）；
- POS 设备通常需 **PCI 批准**；
- 必须使用 **EMVCo 认证（qualified）的 L3 测试工具**（如 FIME BTT、Payhuddle Tecto 等）；
- 每个卡组织自行定义测试计划，并**单独提交认证**——不存在统一的"EMV L3 证书"。

## 二、各卡组织详解

### Visa — ADVT / CDET / Visa L3 Testing

- **ADVT（Acquirer Device Validation Toolkit）**：验证 **接触式芯片** 受理，检查终端配置正确性。
- **CDET（Contactless Device Evaluation Toolkit）**：验证 **非接（NFC）** 支付流程；另有 **qVSDC DM** 工具（条件性必需）。
- **强制要求**：收单机构在终端首次部署前（含所有硬件/软件/参数变体）**必须** 使用 ADVT、CDET、qVSDC DM（如适用）。未使用且引发互操作问题者，可能依 **Visa Chip Interoperability Compliance Program** 被处以罚款。
- **CCRT（Chip Compliance Reporting Tool）**：集中式服务器端工具，用于系统化提交与管理 ADVT/CDET/qVSDC DM 测试结果。
- **前提**：仅可在 EMVCo 批准的终端上执行;新部署须含当前有效 EMVCo 批准的 IFM/内核。
- Visa 通过 Digital Partner Services / Technology Partner 门户提供 L3 测试材料与工具包。

### Mastercard — M-TIP

- **M-TIP（Mastercard Terminal Integration Process）**：覆盖接触 + 非接。
- **TSE（Test Selection Engine）**：Windows 应用（Mastercard 自有组件，可从 Mastercard Connect 获取）。收单机构/VAR 用 TSE 描述终端配置，生成 **唯一 M-TIP 参考号** 和对应测试计划。
- **L3 Test Sets**：M-TIP 测试场景按需求变更/互操作发现不定期更新，以"L3 Test Set"形式由测试工具厂商或 Mastercard Connect 提供。
- **非接前提**：非接 M-TIP 测试需先取得 **Mastercard Contactless Vendor Product Letter of Approval（VPLA）**。
- 工具：可使用 EMVCo 认证的 L3 测试工具（见 EMVCo 官网 L3 Test Tools 列表）。
- 咨询：L2 认证实验室相关问题 → chip_certification_ad@mastercard.com；技术资源见 Mastercard Connect → Technical Resources Center。

### American Express — AEIPS（接触）+ Expresspay（非接）

- **认证程序**：Global test plan（接触/非接）、**Global AFD**（自动加油机）、**Transit**（交通）。
- **支持规范版本**（参考认可工具能力）：AEIPS 6.4、Expresspay 3.1-1.6 / 4.0-2.0、Expresspay Transit 4.0、Amex AFD 1.1 等。
- **前提**：所有 POS 设备需 **PCI 批准**；必须使用 **EMVCo L3 认证测试工具**。
- **ASP（Approved Solution Provider）Program**：Amex 维护认可的收单测试工具供应商清单（FIME 等在列）。

### Discover / Diners Club / PULSE — DPAS L3

- **D-PAS（Discover Payment Application Specification）**：Discover Global Network 的 EMV 规范，覆盖线上/线下、接触/非接交易；同时适用 Discover、Diners Club International 及全球合作网络发卡的卡。
- **DPAS L3（E2E）**：验证终端↔收单主机端到端集成，含授权请求/响应、在线/离线交易、网络专属规则合规。
- 测试材料含 **Fallback 测试卡** 等专项卡片集（如 FIME SAVVI DGN DPAS L3 Fallback Cards）。
- 已有工具实现 Diners 测试计划自动化以加速认证。

### JCB — TCI（Terminal Check for Implementation）

- **TCI** = 接触测试；**TCI-CL** = 非接测试（JCB Contactless / J/Speedy）。
- **定位**：JCB 自有的终端验证/质量保证服务，**收单机构在 JCB 商户部署 IC 终端前必须完成**，以避免互操作问题。
- **前提**：终端须已按 EMVCo 标准完成 L1/L2 验证。JCB 另发布自有 JCB Contactless（L2）IC 终端规范。
- 非接需申请 JCB Contactless IC Terminal Type Approval 初始文档（向 JCB 提交所需信息，通常 3 个工作日内响应）。

### UnionPay International（银联国际）— UAC + QuickPass

- **UAC（UnionPay contact IC card / IC Card）**：接触式 IC 卡终端集成测试。
- **QuickPass（闪付）**：非接测试。
- **两类测试计划**：**Integration（集成）** 与 **Functional（功能）** 测试计划均需完成。
- **测试卡 / 参数**：QuickPass 测试指南为收单机构提供测试卡集（约 17 张模拟不同场景），含功能、离线、在线测试，非接限额（contactless limit）设为 300。
- 工具：FIME Savvi Test Platform 等已获 UPI 认证，可自动化执行 UAC + QuickPass 全部测试用例并管理认证。

## 三、关键要点

- **"一次测试、多处提交"是误区**——虽可用单一工具（如 BTT）驱动，但每个卡组织有**独立的程序、参考号、报告通道**（Visa→CCRT、Mastercard→TSE/M-TIP 号），需分别提交。
- **非接往往有额外前置审批**——Mastercard 需 VPLA、JCB 需 Contactless Type Approval。
- **报告/管理工具是各家强制环节**：Visa 的 CCRT、Mastercard 的 TSE 生成的 M-TIP 编号是认证可追溯性的核心。
- **场景化子计划**（AFD 加油、Transit 交通）针对特定受理环境单独测试，部署这些场景的商户需额外覆盖。
- **本地卡组织**（BankAxept、EFTPOS、ELO、Interac、PayNet 等）有各自 L3 要求，BTT 亦支持，但本文聚焦 6 大国际卡组织。

# [EMV Contactless Kernel Deep Dive](https://smartcardfyi.com/guide/emv-contactless-kernel/)

深入探讨了EMV非接触式支付规范中的核心组件——**内核**。

以下是该网页的核心内容总结：

### 1. 核心概念：什么是内核？
在EMV非接触式支付规范中，**内核** 是支付技术栈中对应于特定支付品牌（如Visa、万事达）的专用逻辑层。它明确定义了支付终端必须发送的APDU命令、必须处理的数据对象以及必须支持的持卡人验证方法。

### 2. 主要内核映射
网页提供了一个内核映射表，列出了主要的7个内核及其对应的支付品牌和应用标识符前缀，例如：
- **Kernel 2**: 万事达（Mastercard Contactless）
- **Kernel 3**: 维萨（Visa Contactless）
- **Kernel 6**: 发现卡/大来卡（Discover/Diners）
- 其他还包括美国运通、JCB、银联等。
为了实现全球受理，一台支付终端通常至少需要实现Kernel 2、3和6。

### 3. 通用预处理层：入口点
在调用任何特定内核之前，终端会先运行**入口点**。这是一个公共的预处理层，主要负责：
- 构建候选应用列表。
- 执行组合选择，将卡支持的应用与终端配置进行匹配。
- 确定由哪个内核来处理交易。

### 4. 两大主要内核的流程详解
网页重点对比了Kernel 2和Kernel 3的交易流程：
- **Kernel 2（万事达）**：采用 **qChip** 流程，偏好在线授权。其核心步骤包括获取处理选项、读取记录、生成授权请求密文，并最终将ARQC发送给发卡行进行在线授权。
- **Kernel 3（维萨）**：支持两种模式：
    - **qVSDC**：完整的芯片交易流程，安全性更高。
    - **MSD**：磁条数据模拟模式，速度更快但安全性较低。

### 5. 持卡人验证方法要求
网页说明了不同内核对于**高额交易**的验证要求：
- **Kernel 2**：需要**在线PIN**或**消费者设备验证**。
- **Kernel 3**：需要**签名**或**在线PIN**。
- **低额交易**（低于“交易限额”）通常无需验证。**CDCVM**（如手机的面部识别、指纹）使得在高于限额的交易中无需在终端输入PIN成为可能。

### 6. 交易限额与风险管理
**发卡行**会为终端配置针对不同支付方案的**交易限额**：
- 交易金额**低于**该限额时，可以在卡上离线完成（生成交易证书）。
- 交易金额**高于**该限额时，则**必须**生成ARQC，并由发卡行在线授权。

### 总结
这份指南系统地解析了EMV非接触式支付中不同品牌内核的工作原理、处理流程及安全要求，主要面向支付系统开发者、技术人员和对支付技术底层逻辑感兴趣的专业人士。网页还提供了相关工具的链接，如**APDU命令构建器**，用于测试相关命令。

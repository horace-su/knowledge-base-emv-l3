# DE55 标签分组速查（必填 / 条件 / 专有 / 响应侧）

> DE55（ISO 8583 **Field 55 = ICC System Related Data**）承载一串 **BER-TLV 编码的 EMV 数据对象**，是发卡行做联机授权与 ARQC 验证的依据。本文把常见标签按**方向与必选度**重新分组，作为速查，并沉淀一次对"必填/可选"常见误判的**勘误**。承接 [DE55 跨卡组织要求（框架篇）](./ISO8583-字段55-跨卡组织要求.md)、[DE55 逐标签实现清单（实现篇）](./ISO8583-DE55-逐标签实现清单.md)、[TAC/IAC/TVR 决策逻辑](../01-基础概念/TAC-IAC-TVR决策逻辑.md)、[APDU/TLV 实测走读](./APDU-TLV实测交易流程解读.md)。
>
> ⚠️ 贯穿性前提：DE55 的"必填/可选"**没有普适答案**，取决于**卡组织 + 交易路径**（联机/离线、CDA/SDA、有无 CDCVM、冲正/退款）。内核只打包"当前存在的"标签，最终以**所对接卡组织主机接口规范**为准。

---

## 一、勘误：对"必填/可选"的六处常见误判

| # | 常见误判 | 修正 |
|---|----------|------|
| ① | 把 `9F33`(Terminal Capabilities) 当作"构建 ARQC 所必需" | `9F33` **不参与 ARQC 密文计算**，是终端能力上报标签；ARQC 输入是 CDOL1 默认元素（`9F02 9F03 5F2A 9A 9C 9F1A 95 9F37` + AIP/ATC）。判据应为"发卡行联机验密文所需"而非"打包 ARQC" |
| ② | 请求侧与响应侧标签混排 | `8A`/`91`/`71`/`72`/`DF31` 是**发卡行→终端方向**（响应/完成报文），须单列 |
| ③ | `9F6E` 直接标为 "Form Factor Indicator" | 仅 **Visa** 成立；`9F6E` 是 **AID-aware 复用**：Visa=FFI / Mastercard=Third Party Data / Amex 另有用法（见[实现篇 §四](./ISO8583-DE55-逐标签实现清单.md)） |
| ④ | `5A`/`57`（PAN/Track2）默认必带 | 很多收单主机**不在 DE55 放明文 PAN/二磁道**（PAN 走 DE2、Track2 走 DE35，PCI 去重）；是否入 55 因主机而异 |
| ⑤ | `DF31` = 脚本结果 | `DFxx` 为**专有域**；EMV 标准的 Issuer Script Results 是 **`9F5B`**，`DF31` 只是某主机私有编码 |
| ⑥ | `9F34`(CVM Results) 列为"可选" | 多数卡组织为**条件必填**（有 CVM 处理即须上报） |

## 二、修订后分组

### A. 联机请求必填（发卡行验密文所需）
| 标签 | 名称 | 来源 |
|------|------|------|
| `9F26` | Application Cryptogram (ARQC) | 卡（GENERATE AC） |
| `9F27` | Cryptogram Information Data | 卡 |
| `9F10` | Issuer Application Data | 卡 |
| `9F36` | Application Transaction Counter (ATC) | 卡 |
| `9F37` | Unpredictable Number | 终端 |
| `95`   | Terminal Verification Results (TVR) | 终端 |
| `82`   | Application Interchange Profile (AIP) | 卡 |

### B. ARQC 输入（CDOL1 默认元素，随请求送）
| 标签 | 名称 |
|------|------|
| `9F02` | Amount, Authorized |
| `9F03` | Amount, Other |
| `5F2A` | Transaction Currency Code |
| `9A`   | Transaction Date |
| `9C`   | Transaction Type |
| `9F1A` | Terminal Country Code |

> A+B 即框架篇/实现篇的"联机批准"实测请求集：`9F26 9F27 9F10 9F36 9F37 95 82 9F1A 5F2A 9F02 9C 9A`（**不含 `9F33`**）。

### C. 终端/卡上报（常必带，非 ARQC 输入）
| 标签 | 名称 |
|------|------|
| `9F33` | Terminal Capabilities |
| `9F35` | Terminal Type |
| `9F1E` | Interface Device (IFD) Serial Number |
| `9F09` | Application Version Number |
| `9F41` | Transaction Sequence Counter |
| `84`   | Dedicated File (DF) Name |
| `5F34` | Application PAN Sequence Number（外层亦映射 DE23，见[路径对照篇](./ISO8583-磁条-接触-非接报文差异对照.md)） |

### D. 条件必填
| 标签 | 名称 | 条件 |
|------|------|------|
| `9F34` | CVM Results | 有 CVM 处理即须上报（见 [接触与非接 CVM 详解](../01-基础概念/接触与非接CVM详解.md)） |

### E. 卡组织专有（AID-aware，须按选中 AID 决定语义）
| 标签 | 名称 | 归属 |
|------|------|------|
| `9F6E` | FFI / Third Party Data | **Visa=FFI**；MC=Third Party Data；Amex 另有用法 |
| `9F66` | Terminal Transaction Qualifiers (TTQ) | Visa（见 [Visa TTQ/CTQ](../04-visa专题/Visa-TTQ-CTQ与CDCVM-Token化指示.md)） |
| `9F6C` | Card Transaction Qualifiers (CTQ) | Visa |
| `9F1D` | Terminal Risk Management Data | Mastercard |
| `9F53` | Terminal Interchange Profile (TIP) | **JCB**（通用语义下 `9F53`=Transaction Category Code，复用冲突，须按 AID 判定） |

### F. PCI 敏感 / 主机决定是否入 55
| 标签 | 名称 | 说明 |
|------|------|------|
| `5A`  | Application PAN | 多数主机不在 55 放明文 PAN（走 DE2） |
| `57`  | Track 2 Equivalent Data | 多数主机不在 55 放二磁道（走 DE35） |

### G. 响应 / 完成侧（发卡行 → 终端）
| 标签 | 名称 | 说明 |
|------|------|------|
| `91`   | Issuer Authentication Data (ARPC) | 发卡行下发 |
| `8A`   | Authorization Response Code (ARC) | 发卡行下发（响应码语义见 [DE39 应答码篇](./ISO8583-DE39应答码与拒绝治理.md)） |
| `71`   | Issuer Script Template 1 | 发卡行下发，GENERATE AC **之前**执行 |
| `72`   | Issuer Script Template 2 | 发卡行下发，GENERATE AC **之后**执行 |
| `DF31` | Issuer Script Results（专有） | 终端执行后回送；标准等价 **`9F5B`** |

## 三、速记

1. **"打包 ARQC" ≠ "DE55 必填"**：进密文的是 CDOL1 元素，终端能力标签（`9F33/9F35`）另走上报。
2. **分方向看**：请求侧（卡/终端产出）与响应/完成侧（发卡行下发的 `91/8A/71/72`、终端回送的脚本结果）不要混。
3. **AID-aware 解析**：`9F6E`、`9F53` 语义随卡组织变，主机解析器不能一刀切。
4. **PAN/Track2 慎入 55**：多数主机走 DE2/DE35，DE55 内是否携带看接口规范。
5. 精确必选集仍以**各卡组织主机接口规范**最新版为准。

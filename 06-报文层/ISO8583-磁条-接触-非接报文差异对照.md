# ISO 8583 报文差异对照：磁条 / 接触芯片 / 非接芯片

> 同一笔卡支付，在**磁条刷卡（swipe）/ 接触芯片插卡（contact）/ 非接拍卡（contactless）** 三种受理路径下，外层 ISO 8583 报文的差异。核心结论：**真正区分三者的不是 MTI，而是 DE22（录入方式）+ DE55（有没有 EMV TLV）+ Track 域（DE35/DE45）+ 卡片派生域（DE23↔`5F34`）这一组域是否自洽**。
>
> 承接 [ISO 8583 报文域全景与 POS 录入方式（DE22）/ 技术回退](./ISO8583-报文域全景与POS录入方式.md)（DE22 子域与回退细节看那篇）、[DE55 框架篇](./ISO8583-字段55-跨卡组织要求.md)、[DE55 逐标签实现清单](./ISO8583-DE55-逐标签实现清单.md)、[APDU/TLV 实测交易流程解读](./APDU-TLV实测交易流程解读.md)；CVM 侧看 [接触与非接 CVM 详解](../01-基础概念/接触与非接CVM详解.md)、[EMV 免密免签判断流程](../01-基础概念/EMV免密免签判断流程.md)。
>
> ⚠️ 各域精确码值/长度/子域随报文协议与卡组织而异（Visa BASE I/II、Mastercard CIS/IPM、银联 8583、各收单主机手册）。本文给出业界通行框架与 Mastercard 取向，精确取值以**你对接的卡组织/收单主机报文规范最新版**为准。

## 一、总览对照表

| 报文域 | 磁条卡（swipe） | 接触芯片（contact / 插卡） | 非接芯片（contactless / 拍卡） |
|--------|----------------|---------------------------|-------------------------------|
| **DE22 SF1（录入方式）** | `90` 磁条整轨（服务码可信）/ `02` | **`05` 芯片读取成功** | **`07` 非接 EMV 自动录入** |
| **DE55（ICC TLV）** | **不带**（磁条无 EMV 内核产物） | **必带**：`9F26` ARQC、`9F27` CID、`82` AIP、`95` TVR、`9F36` ATC、`9F10` IAD… | **必带**，且含非接专有项：`9F66`(TTQ,Visa)/`9F6E`(FFI,MC)、`9F6C`(CTQ)、限额判定等 |
| **DE2 PAN** | 来自 Track2（`5A` 等效） | 来自芯片 `5A` | 来自芯片 `5A`；**Token 卡为 DPAN** |
| **DE23 卡序列号** | **基本不带 / `000`**（磁道无此字段） | 卡有 `5F34` 则带，须 = `5F34` | 同接触，卡有 `5F34` 则带 |
| **DE35 Track2** | **主数据来源**，真实磁道 | **芯片等效 Track2**（`57`） | 同左（`57` 等效） |
| **DE45 Track1** | 真实磁道（视读卡能力） | 可选（芯片很少给 Track1） | 一般不带 |
| **服务码（Track 内 SSS）** | 芯片卡应为 `2xx/6xx`(请用芯片) → 标 `90` 会与发卡行冲突 | `2xx/6xx`，与 `05` 自洽 | 同左 |
| **DE52（在线 PIN 块）** | 有 PIN Pad 且做 Online PIN 才带 | 视 CVM：Online PIN 才带 | 多为 No CVM/CDCVM → **通常不带** |
| **CVM 体现** | 签名 / Online PIN | `9F34` CVM Results（脱机 PIN/签名/联机 PIN） | `9F34` + 限额判定，常 No CVM 或 CDCVM |
| **DE61 SF11 / `9F33`** | 仅磁条读能力 | 接触接口能力 | 非接接口能力 |

## 二、三点实质区别

### 2.1 有没有 DE55 = 芯片 vs 磁条的根本分水岭
磁条交易没有 EMV 内核参与，发卡行只能凭磁道数据 + CVV 做**静态风控**；芯片（接触/非接）都跑了内核，产出**动态密文 `9F26`(ARQC)** 装进 DE55，发卡行做联机密文校验（动态认证）。安全等级完全不同——这也是各卡组织 fallback 限频、对纯磁条收紧的根因。

### 2.2 接触 vs 非接 = DE22 码值 + DE55 内的非接专有标签
- **DE22**：`05`（接触）vs `07`（非接）。
- **DE55 专有项**：非接会多出内核交互产物——Visa 的 `9F66`(TTQ)、`9F6C`(CTQ)，Mastercard（Kernel 2）的 `9F6E`(FFI)、四大限额判定结果、RRP 相关；接触交易没有这些。内核编号见 K2=Mastercard、K3=Visa、K6=Discover（详见 [非接内核 Deep Dive](../01-基础概念/emv-contactless-kernel-deep-dive.md)）。
- **CVM**：非接更常走 **No CVM（小额免签）或 CDCVM（手机端验证）**，DE52 多半不带；接触更常见脱机/联机 PIN。

### 2.3 还要分清两类"降级"，别和正常路径混淆

```
05  接触芯片读成功        ┐ 正常 EMV，带完整 DE55
07  非接芯片读成功         ┘
─────────────────────────
80  芯片读失败→回退磁条    ← 技术回退！必须标 80，不是 90；DE55 缺位或仅子集
90  纯磁条卡刷卡           ← 卡本就没芯片才对
91  非接降级为磁道(MSD)    ← 非接侧降级，不是接触回退
```

- **`80`（technical fallback）**：芯片卡在支持芯片的终端上读失败后退到磁条。和 `90` 的区别是 L3 硬要求——芯片卡服务码 `2xx/6xx`(请用芯片)，标 `90` 与服务码自相矛盾会被风控/拒绝。逐字段实测失败案例见 [DE22 篇 §七 M-TIP02](./ISO8583-报文域全景与POS录入方式.md#七实测案例m-tip02技术回退失败de2290应为80)。
- **`91`（contactless MSD）**：非接接口降级为磁道数据，又是另一回事，不要与接触回退 `80` 混。

## 三、逐域差异详解

### 3.1 DE22（录入方式）
线上 DE22 通常是 **SF1 + SF2 拼成的 3 位** `n3`：抓包看到 `051`(接触+可输PIN)、`071`(非接+可输PIN)、`901`(磁条+可输PIN)、`801`(回退)，而非孤立的 `05/07`。SF2 是终端 PIN 能力位（`1` 有 / `2` 无 / `8` 故障），须与 `9F33` CVM Capability、DE52 是否在场自洽。完整码值集与回退触发链见 [DE22 篇 §三/§四](./ISO8583-报文域全景与POS录入方式.md)。

### 3.2 DE55（ICC TLV）
- **磁条**：无 DE55。
- **接触**：通用 EMV 集——`9F26` ARQC、`9F27` CID、`9F36` ATC、`95` TVR、`9F10` IAD、`82` AIP、`9A`/`9F02`/`5F2A`/`9F1A` 等。
- **非接**：在通用集之上叠加内核/品牌专有标签（见 2.2）。
- 逐标签字节级清单与各卡组织差异见 [DE55 实现篇](./ISO8583-DE55-逐标签实现清单.md) 与 [DE55 框架篇](./ISO8583-字段55-跨卡组织要求.md)。

### 3.3 DE2 / DE35 / DE45 与服务码
- **磁条**：DE35/DE45 是**真实磁道**，DE2 从 Track2 解析。
- **芯片（接触/非接）**：DE35 来自芯片 `57`（Track2 Equivalent Data），DE2 来自 `5A`；DE45 多数芯片不提供。
- **服务码**：磁道服务码首位 `2`/`6` = "有芯片则必须用芯片"，是反推卡是否芯片卡、校验 DE22 标得对不对的依据（详见 [DE22 篇 §五](./ISO8583-报文域全景与POS录入方式.md#五de35--de45-轨道数据与服务码)）。

### 3.4 DE52 / CVM
- **磁条**：签名或 Online PIN（带 DE52）。
- **接触**：CVM 由 CVM List(`8E`) 决策，结果在 `9F34`；走 Online PIN 才带 DE52。
- **非接**：常 No CVM / CDCVM，DE52 通常缺位；CDCVM 在 `9F34` 与 TTQ/CTQ/FFI 指示位体现。CVM 决策链见 [接触与非接 CVM 详解](../01-基础概念/接触与非接CVM详解.md)、[免密免签判断流程](../01-基础概念/EMV免密免签判断流程.md)。

## 四、DE23（卡序列号 / PAN Sequence Number）专节

> 这是把本篇单列出来的直接触发点——DE23 是"芯片卡才真正有"的域，三路径差异的一个典型缩影。

- **定义**：DE23 = Card Sequence Number，格式 `n3`，对应 EMV 标签 **`5F34`**。作用是区分**同一 PAN 下的多张物理卡/多枚芯片**（主卡+附属卡、换卡补卡同号不同序），让发卡行定位到具体哪一张。

| 路径 | DE23 来源 | 是否带 |
|------|----------|--------|
| 磁条卡 | 磁道里**没有** PAN Seq 字段 | **基本不带 / 送 `000`** |
| 接触芯片 | 芯片 `5F34`（READ RECORD 阶段） | **卡有就带**；卡未个人化 `5F34` 则不带或 `000` |
| 非接芯片 | 芯片 `5F34`（GPO/READ RECORD） | 同接触；Token 的 DPAN 也可能带 |

要点：
- **`5F34` 是可选标签**——不是每张芯片卡都个人化了它。卡里没有，DE23 就不送或送 `000`，属正常，不是错误。
- **DE23 ↔ `5F34` 必须一致**：这是 L3 报文层跨域自洽校验之一（与 DE2↔`5A`、DE4↔`9F02`、DE49↔`5F2A` 同类）。芯片交易 DE55 里带了 `5F34`，外层 DE23 必须等于它，不一致判失败。
- **磁条/回退（`80`）交易**：没读出芯片 `5F34`，DE23 缺位或 `000`——与"磁条无此字段"一致，不矛盾。
- **格式细节**：`5F34` 在芯片里是 1 字节（如 `01`），映射到 DE23 的 `n3` 时按数值补零成 3 位（→ `001`）。有的卡个人化为 `00`、有的从 `01` 起，取决于发卡行。

## 五、三路径报文骨架（域级示意）

> 仅示意"带哪些域"，非逐字节。完整一笔交易的 APDU↔TLV↔DE55 字节级走读见 [APDU/TLV 实测交易流程解读](./APDU-TLV实测交易流程解读.md)。

```
磁条 swipe（0100/0200）
  MTI | DE2(PAN) | DE3 | DE4 | DE11 | DE14 | DE22=901 | DE35(真实Track2) | [DE45] | [DE52] | DE61
  ——无 DE55、无 DE23(或000)

接触芯片 contact（0100/0200）
  MTI | DE2 | DE3 | DE4 | DE11 | DE14 | DE22=051 | DE23(=5F34，若卡有) | DE35(57等效) |
      [DE52] | DE55(9F26/9F27/95/9F36/9F10/82…) | DE61
  ——动态密文走 DE55

非接芯片 contactless（0100/0200）
  MTI | DE2(或DPAN) | DE3 | DE4 | DE11 | DE14 | DE22=071 | DE23(若卡有) | DE35(57等效) |
      DE55(通用 + 9F66/9F6E/9F6C 等非接专有) | DE61
  ——多数 No CVM/CDCVM，常无 DE52
```

## 六、L3 报文层校验清单（按路径）

| 校验项 | 磁条 | 接触 | 非接 |
|--------|------|------|------|
| DE22 SF1 录入方式正确 | `90`（非芯片卡） | `05` | `07` |
| DE55 是否在场 | 不应有 | **必须有，含 `9F26`** | **必须有，含非接专有标签** |
| DE23 ↔ `5F34` 一致 | 缺位/`000` | 卡有则须等 `5F34` | 卡有则须等 `5F34` |
| DE35/DE45 与服务码自洽 | 真实磁道，服务码可信 | `57` 等效，服务码 `2xx/6xx` | 同接触 |
| DE52 / CVM 自洽 | 签名/Online PIN | `9F34` 与 DE52/`9F33` 自洽 | 多 No CVM/CDCVM，慎带 DE52 |
| DE22 SF2 PIN 能力 | 与 `9F33`、DE52 自洽 | 同 | 小额非接常 SF2=2 |
| 降级路径标志 | — | 芯片失败回退须 `80`（非 `90`） | 非接降磁道为 `91` |
| 跨域一致 | DE2↔Track | DE2↔`5A`、DE4↔`9F02`、DE49↔`5F2A`、DE14↔Track 有效期、DE23↔`5F34` | 同接触 |

# ISO 8583 字段 55（DE55）与各卡组织 EMV 数据要求

> L3 主机侧测试的核心：终端把内核产出的 EMV 数据打包进 ISO 8583 **Field 55（DE55）** 送往发卡行/卡组织，各卡组织对 DE55 内含哪些 EMV 标签、必选/可选、长度与格式的要求**并不相同**。本文梳理 DE55 的来源（内核 Data Record）、通用必备标签，以及各卡组织差异。承接 [TAC/IAC/TVR 决策逻辑](../01-基础概念/TAC-IAC-TVR决策逻辑.md)、[ODA 证书链字节级](../01-基础概念/ODA证书链字节级.md)、[EMVCo Kernel 2 V2.11 规范要点](../01-基础概念/EMVCo-Kernel2-V2.11规范要点.md)、[L3 认证实例：Sunmi T6F10](../07-实测案例/L3认证实例-Sunmi-T6F10终端配置剖析.md)。

---

## 一、DE55 是什么

- ISO 8583 中 **Field 55 = "Integrated Circuit Card (ICC) System Related Data"**，承载一串 **BER-TLV 编码的 EMV 数据对象**。
- 内容**不是终端随意填**：接触交易由 EMV 内核在 GENERATE AC 后产出，非接由各卡组织内核（Kernel 2/3/6…）在交易完成时通过 **Outcome + Data Record** 输出（见 [Kernel 2 要点](../01-基础概念/EMVCo-Kernel2-V2.11规范要点.md)）。
- DE55 是发卡行做**联机授权**与**ARQC 验证**的依据，因此 L3 主机测试要校验：标签齐全、值正确、TLV 编码合法。

> 实例佐证：[Sunmi T6F10 报告](../07-实测案例/L3认证实例-Sunmi-T6F10终端配置剖析.md) 的 M-TIP 问卷里就有 **"Additional Tags supported in DE55 = None"** 与 **"Terminal Input Capability Indicator (DE61 SF11) = 3"**——即 DE55 的附加标签集是 L3 报备项之一。

---

## 二、DE55 的来源：内核 Data Record

Mastercard Kernel 2 的 **Data Record（`FF8105`）** 明确定义了交易完成时随 Outcome 返回的 TLV 清单（EMV 模式，Book C-2 Table 4.6）——这正是 DE55 的"原料表"：

| 类别 | 数据对象（标签） |
|------|------------------|
| 金额/交易 | Amount Authorized `9F02`、Amount Other `9F03`、Transaction Currency Code `5F2A`、Transaction Date `9A`、Transaction Type `9C`、Transaction Category Code `9F53` |
| 密文/风控 | Application Cryptogram `9F26`、Cryptogram Information Data `9F27`、Issuer Application Data `9F10`、Application Transaction Counter `9F36`、Unpredictable Number `9F37`、**TVR `95`** |
| 应用/卡 | AIP `82`、Application PAN `5A`、PAN Seq `5F34`、AID/DF Name `84`、App Version Number `9F09`、Application Usage Control `9F07`、App Expiration Date `5F24`、Track 2 Equivalent `57` |
| 终端 | Terminal Capabilities `9F33`、Terminal Country Code `9F1A`、Terminal Type `9F35`、IFD Serial `9F1E` |
| CVM/令牌 | CVM Results `9F34`、Payment Account Reference(PAR) `9F24`、Token Requestor ID `9F19`（v2.11 新增） |

> 内核只把"**当前存在的**"标签加入 Data Record（`IF IsPresent(Tag) THEN AddToList`）。所以 DE55 实际内容随交易路径（联机/离线、CDA/SDA、CDCVM 有无）动态变化——这也是 L3 用例要覆盖多场景的原因。

## 三、通用"必备"EMV 标签（多数卡组织联机授权都要）

| 标签 | 名称 | 为何必备 |
|------|------|----------|
| `9F26` | Application Cryptogram (ARQC) | 发卡行验密文的核心 |
| `9F27` | Cryptogram Information Data | 指示 AAC/ARQC/TC |
| `9F10` | Issuer Application Data | 发卡行私有风控数据 |
| `9F37` | Unpredictable Number | 防重放 |
| `9F36` | Application Transaction Counter | 防克隆/计数 |
| `95` | TVR | 终端验证结果（见 [TAC/IAC/TVR](../01-基础概念/TAC-IAC-TVR决策逻辑.md)） |
| `9F1A` | Terminal Country Code | |
| `5F2A` | Transaction Currency Code | |
| `82` | AIP | |
| `9F33` / `9F35` / `9F34` | Terminal Capabilities / Type / CVM Results | |
| `9F02` / `9C` / `9A` | Amount / Transaction Type / Date | |

## 四、各卡组织 DE55 要求差异（要点）

> ⚠️ 下表为**框架性差异**，精确的必选/可选标签集与长度以**各卡组织主机接口规范**（Visa Field 55 / Mastercard CIS / Amex GCAG / Discover/JCB/UPI 报文手册）最新版为准。

| 卡组织 | DE55 / 等效字段 | 标志性差异点 |
|--------|------------------|--------------|
| **Visa** | Field 55（VSDC）；qVSDC 非接 | 强调 `9F6E`(FFI)/`9F66`(TTQ)/`9F6C`(CTQ) 等 Visa 专有标签（见 [Visa TTQ/CTQ](../04-visa专题/Visa-TTQ-CTQ与CDCVM-Token化指示.md)）；Token 交易带 `9F24` PAR |
| **Mastercard** | DE55；CIS 接口 | M/Chip 强制 `9F10` IAD（含 CVN/计数器）；非接 `9F1D` 终端风险管理数据；"Additional Tags in DE55" 在 M-TIP 中可选报备 |
| **Amex (AEIPS/Expresspay)** | GCAG ISO/XML 报文 | 走 GCAG 报文模型，[报告示例](../07-实测案例/L3认证实例-Sunmi-T6F10终端配置剖析.md) 中 GCAG ISO **不支持离线/在线退款**，DE55 标签集随 GCAG 版本定 |
| **Discover (D-PAS)** | Field 55；可选 Field 55 Tags | [报告](../07-实测案例/L3认证实例-Sunmi-T6F10终端配置剖析.md) 明确"Field 55 Optional Tags 是否支持/与发卡行共享"为报备项；反转(X400/X420)报文中 Field 55 是否带也单列 |
| **JCB (J/Smart)** | Field 55 | 复用标准 EMV 标签 + JCB 专有（如 `9F53` Terminal Interchange Profile） |
| **UnionPay (QuickPass)** | Field 55；DF21 CVM Limit | 国密交易用 SM 算法密文；TVR/TSI 输出须采集报送（见 [银联国际 QuickPass](../03-各卡组织L3认证/银联国际-QuickPass-L3配置与HK-SG特殊CVM.md)）；国内 PBOC 见 [PBOC 3.0 与国密](../03-各卡组织L3认证/银联国内-PBOC3.0与国密算法.md) |

## 五、L3 主机测试关注点

1. **TLV 合法性**：DE55 必须是合法 BER-TLV，标签顺序/长度/嵌套模板正确。
2. **场景覆盖**：联机批准/拒绝、ARQC→ARPC→TC、CDA vs SDA、CDCVM 有无、退款/冲正——每条路径下 DE55 标签集不同（因内核只放 present 的标签）。
3. **必备标签齐全**：缺 `9F26`/`9F27`/`9F10`/`9F37` 等会直接导致发卡行无法验密文。
4. **卡组织专有标签**：Visa `9F6E`、MC `9F1D`、JCB `9F53` 等需按各自规范出现。
5. **DE61/DE55 关联**：终端能力指示（如 DE61 SF11）须与 DE55 内 `9F33`/`9F35` 自洽。

> 抓包核对可用 Card Spy / SmartSpy+ 解析真实 APDU 与 DE55 TLV（见 [FIME-BTT](../02-fime测试工具/FIME-BTT-品牌测试工具深度解析.md)）。
